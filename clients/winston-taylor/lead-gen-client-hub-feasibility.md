# Lead Gen / Client Hub Feasibility

**Overview** Explore feasibility of a gated content and lead qualification system, with a longer-term view toward a client hub, for inclusion in or alongside the MVP.

---

**Feature 1: Gated Content**

- Publish a teaser/preview of key reports (e.g. PE report) publicly
- Gate the full content behind a form to capture user details
- Allow gate position to be tested and moved (already proven to affect results)
- Must support multiple content types (reports, thought leadership, insights)

**Feature 2: Lead Capture & BD Workflow**

- Captured form data should feed into a CRM or BD tool automatically
- BD team reviews and qualifies leads before passing to lawyers
- Avoid cold outreach — build in a consent or acknowledgment step so follow-up feels bespoke, not intrusive

**Feature 3: Content Performance Tracking**

- Track which gated content gets the most downloads
- Identify patterns in user type or sector engaging with specific content
- Surface this data to BD and content teams to inform follow-up and future content decisions

**Feature 4: Visitor Intelligence (longer-term / post-MVP)**

- Leverage existing IP/Hotjar/heat map data to identify company-level visitors
- Feed signals into BD pipeline

**Feature 5: Client Portal Access Point (vision only)**

- Explore feasibility of website acting as an authenticated entry point to client matter resources
- Would require API integration with existing portal systems

---

**Notes for Feasibility Assessment**

- Gating is already in use but current implementation is manual and disjointed — an automated, integrated solution would be differentiating
- Comparable approach already used by accountancy firms
- Priority for MVP should be Features 1 & 2; Features 3–5 can be phased
